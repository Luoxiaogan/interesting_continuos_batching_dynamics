# `run_coupling_experiment.py` 运行依赖清单

## 仓库内必需 Python 文件

运行 `coupling_experiments/run_coupling_experiment.py` 现在只依赖 `coupling_experiments/` 目录内的这些 Python 文件：

1. `coupling_experiments/run_coupling_experiment.py`
2. `coupling_experiments/theory_simulator.py`
3. `coupling_experiments/multi_type_simulator.py`
4. `coupling_experiments/metrics.py`
5. `coupling_experiments/visualize_coupling.py`

## 运行时读取的非 Python 文件

1. `coupling_experiments/config.json`

## 第三方 Python 包

这些依赖仍然需要通过 Python 环境安装，它们不属于仓库内代码复制范围：

1. `numpy`
2. `scipy`
3. `matplotlib`

## 标准库

脚本还使用 Python 标准库，不需要额外安装：

- `os`
- `json`
- `csv`
- `datetime`
- `pathlib`
- `subprocess`
- `math`
- `typing`

## 改造说明

原来 `run_coupling_experiment.py` 会通过 `sys.path` 注入 `../new_project_for_multi_type`，再从那里导入 `multi_type_simulator.py`。

现在已经把该文件复制到 `coupling_experiments/multi_type_simulator.py`，并删除跨目录 `sys.path` 依赖；因此仓库内的 Python 代码依赖已经全部收拢到 `coupling_experiments/` 内。
